# Bonkhemist i18n (ES/EN) — Design

**Date:** 2026-06-24
**Scope:** `bonkhemist-front` only. No backend changes.
**Branch:** `feat/i18n-es-en`

## Goal

Add Spanish/English internationalization across the entire static UI of the
Bonkhemist frontend, with a simple in-place language toggle. AI-generated
element content (names, descriptions, reasoning) is out of scope — it keeps
coming from the backend in English.

## Decisions (agreed in brainstorming)

| Decision | Choice |
|----------|--------|
| Scope | Static UI only (frontend). Zero backend changes. |
| Routing | Simple toggle, **no URL prefix** (`strategy: 'no_prefix'`). |
| Persistence | Cookie/localStorage via `detectBrowserLanguage.useCookie`. |
| First-visit detection | Browser language → `es` if Spanish, else fallback. |
| Base / fallback language | **English** (`en`). Matches current hardcoded copy 1:1. |
| Library | `@nuxtjs/i18n` **v8** (compatible with Nuxt 3.10.2). |

## Architecture

### Module configuration (`nuxt.config.ts`)

Add `@nuxtjs/i18n` to `modules` and configure:

```ts
i18n: {
  strategy: 'no_prefix',
  defaultLocale: 'en',
  locales: [
    { code: 'en', name: 'English', file: 'en.json' },
    { code: 'es', name: 'Español', file: 'es.json' },
  ],
  langDir: 'locales',
  lazy: true,
  detectBrowserLanguage: {
    useCookie: true,
    cookieKey: 'bonkhemist_lang',
    fallbackLocale: 'en',
  },
}
```

This gives browser detection on first visit, persistence of the user's choice,
and no change to the URL — exactly the agreed behavior.

### Message files & key convention

`locales/en.json` and `locales/es.json`, with keys namespaced by feature:

```
common.*        Loading, Close, Cancel, Confirm, …
header.*        playMusic, challenges, clearElements, howToPlay, searchPlaceholder, elementsCount
howTo.*         title, step1, step2, …
mint.*          modal copy for Mint.vue
mintDiscovery.* modal copy for MintDiscovery.vue
leaderboard.*   table headings, "youCreated": "You created {name}", ranks
challenges.*    Challenges.vue copy
nfts.*          Nfts.vue copy
element.*       Element.vue copy (e.g. "View Reason")
```

- Interpolation uses vue-i18n named syntax: `"youCreated": "You created {name}"`.
- Pluralization (vue-i18n `|` syntax) only where it genuinely applies.
- **Both files MUST contain the exact same set of keys** (enforced by the
  parity check below).

### Language switcher

- New component `components/LanguageSwitcher.vue`: a pixel-art **[ES] [EN]**
  toggle that matches the existing retro styling (Silkscreen/Bungee fonts,
  bordered button look).
- Placed in `components/SiteHeader.vue`, near the hamburger menu / wallet
  button (top-right).
- Switches via `setLocale()`; persistence is automatic via the cookie.

## String extraction (the bulk of the work)

Walk the ~10 files that contain UI copy and replace text only — **no changes to
styles, classes, or logic**:

- Template text: `Play Music` → `{{ $t('header.playMusic') }}`
- Attributes: `placeholder="Search Element..."` → `:placeholder="$t('header.searchPlaceholder')"`
- Script strings (toasts, computed labels): `const { t } = useI18n()` then `t('...')`

Files in scope (by approximate string count from the scan):

| `pages/leaderboard.vue` | 26 |
| `components/Alchemy/HowTo.vue` | 23 |

Out of scope: AI-generated element names/descriptions/reasoning (backend, English).

## Verification

The project has no test runner, so:

1. **Key parity script** `scripts/i18n-check.mjs`: fails (non-zero exit) if
   `en.json` and `es.json` do not contain exactly the same set of keys.
   Prevents missing translations.
2. **Visual verification** in the browser: run the frontend, toggle ES↔EN, and
   screenshot the key screens (home toolbar, HowTo, Mint, MintDiscovery,
   Leaderboard) in both languages to confirm nothing is left untranslated and
   layout holds.

## Deliverables

- `nuxt.config.ts` updated with the i18n module + config.
- `locales/en.json` + `locales/es.json` (complete, key-parity verified).
- `components/LanguageSwitcher.vue`.
- ~10 `.vue` files converted to `$t(...)`.
- `scripts/i18n-check.mjs`.
- All on branch `feat/i18n-es-en`.

## Non-goals

- Translating AI-generated element content.
- URL-prefixed locale routing / per-language SEO.
- Languages other than ES/EN.
- Number/date/currency localization beyond what vue-i18n provides by default.
