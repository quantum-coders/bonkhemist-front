# Bonkhemist i18n (ES/EN) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Spanish/English internationalization to the entire static UI of the Bonkhemist Nuxt frontend with an in-place language toggle.

**Architecture:** Use `@nuxtjs/i18n` (no_prefix strategy) with two JSON message files (`en.json`, `es.json`). Browser language is detected on first visit and the choice is persisted in a cookie. A pixel-art ES/EN switcher in the header flips locale at runtime. Every hardcoded UI string in `.vue` files is replaced with a `$t('namespace.key')` lookup. AI-generated element content stays in English (backend, out of scope).

**Tech Stack:** Nuxt 3.10.2 (SPA, `ssr: false`), Vue 3, `@nuxtjs/i18n` v8, vue-i18n v9, yarn.

## Global Constraints

- Frontend only ŌĆö **no changes to `bonkhemist-api`**.
- `@nuxtjs/i18n` must be **v8.x** (v9 requires Nuxt > 3.10; this repo is 3.10.2).
- Strategy is `no_prefix` ŌĆö the URL never changes.
- Base & fallback locale is `en`. EN values are the existing copy verbatim.
- `en.json` and `es.json` MUST always contain the **exact same set of keys**.
- Keys are namespaced by feature: `common.*`, `header.*`, `howTo.*`, `mint.*`, `mintDiscovery.*`, `leaderboard.*`, `challenges.*`, `nfts.*`, `element.*`.
- Touch **text only** ŌĆö never change styles, classes, structure, or logic.
- Work on branch `feat/i18n-es-en`. Commit after each task.
- Verification per task = (a) `node scripts/i18n-check.mjs` passes, (b) visual toggle ESŌåöEN on the affected screen.

- [ ] **Step 2: Register & configure the module in `nuxt.config.ts`**

Add `'@nuxtjs/i18n'` to the `modules` array and add this top-level key:
```ts
i18n: {
  strategy: 'no_prefix',
  defaultLocale: 'en',
  locales: [
    { code: 'en', name: 'English', file: 'en.json' },
    { code: 'es', name: 'Espa├▒ol', file: 'es.json' },
  ],
  langDir: 'locales',
  lazy: true,
  detectBrowserLanguage: {
    useCookie: true,
    cookieKey: 'bonkhemist_lang',
    fallbackLocale: 'en',
  },
},
```

- [ ] **Step 3: Create seed message files**

`locales/en.json`:
```json
{
  "common": { "loading": "Loading...", "close": "Close", "cancel": "Cancel", "confirm": "Confirm" }
}
```
`locales/es.json`:
```json
{
  "common": { "loading": "Cargando...", "close": "Cerrar", "cancel": "Cancelar", "confirm": "Confirmar" }
}
```

- [ ] **Step 4: Create the key-parity check script**

`scripts/i18n-check.mjs`:
```js
import { readFileSync } from 'node:fs';

const flat = (obj, prefix = '') =>
  Object.entries(obj).flatMap(([k, v]) =>
    v && typeof v === 'object' && !Array.isArray(v)
      ? flat(v, `${prefix}${k}.`)
      : [`${prefix}${k}`]
  );

const en = flat(JSON.parse(readFileSync('locales/en.json', 'utf8')));
const es = flat(JSON.parse(readFileSync('locales/es.json', 'utf8')));
const setEn = new Set(en), setEs = new Set(es);
const missingEs = en.filter(k => !setEs.has(k));
const missingEn = es.filter(k => !setEn.has(k));

if (missingEs.length || missingEn.length) {
  if (missingEs.length) console.error('Missing in es.json:', missingEs);
  if (missingEn.length) console.error('Missing in en.json:', missingEn);
  process.exit(1);
}
console.log(`i18n OK ŌĆö ${en.length} keys in sync.`);
```

- [ ] **Step 5: Verify boot + parity**

Run:
```bash
cd bonkhemist-front && node scripts/i18n-check.mjs && yarn dev
```
Expected: `i18n OK ŌĆö 4 keys in sync.`; dev server boots on the configured port without i18n errors. (Stop the dev server after confirming.)

- [ ] **Step 6: Commit**

```bash
git add package.json yarn.lock nuxt.config.ts locales scripts/i18n-check.mjs
git commit -m "feat(i18n): install @nuxtjs/i18n, scaffold locales + parity check"
```

## Notes for the implementer

- **Reading before extracting:** always read the full `.vue` file first; some strings live in `computed`, `setup`, watchers, or `data`, not just the template.
- **Don't translate:** element names/slugs/descriptions returned by the API, wallet addresses, brand "Bonkhemist", `BKM` symbol, URLs.
- **Spanish length:** Spanish runs ~15-30% longer; if a fixed-width pixel button overflows, that's a real finding ŌĆö note it, don't silently truncate the translation.
- **No test runner exists**; the parity script + browser toggle are the verification. Do not invent a Jest/Vitest setup unless asked.
